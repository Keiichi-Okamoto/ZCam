# AGENTS.md

このファイルは、このリポジトリで Codex を使う時の運用ルール。

## 最初に読むファイル

- 作業開始時はまず `CLAUDE.md` を読む
- あわせて `FEATURES.md` と `PROJECT_CONTEXT.md` を確認する
- `.md` を更新したとユーザーから言われた場合は、必要なファイルを読み直す

## 基本方針

- `CLAUDE.md` のプロジェクト方針・ブランチ方針・ドキュメント方針を尊重する
- 実機確認が必要な事項は、推測で断定せずユーザー確認を前提に進める
- SwiftUI で無理をせず、必要なら `UIViewRepresentable` を使う

## 編集ルール

- 読み取り、調査、`git diff` などの確認系コマンドは確認不要
- `.md` ファイルの追記・更新は確認不要
- ソースコードファイルを書き換える前には、必ずユーザー確認を取る
- シミュレータ / 実機の分岐を増やす場合は、実装前に必ず確認を取る

## ビルド・テスト

- ビルド、テスト、Lint の実行は確認不要
- 実機でしか確認できない項目は、その旨を明記してユーザーに確認を依頼する

## Git / GitHub

- `git commit` はユーザーが明示した時だけ行う
- `git push` はユーザーが明示した時だけ行う
- PR の作成・更新はユーザーが明示した時だけ行う
- 勝手にブランチ作成、push、PR作成をしない

## 期待する役割

- Codex には、局所修正、不具合調査、差分確認、コード読解、設計上の切り分けを優先して依頼する
- 大きな仕様追加でも対応は可能だが、境界が明確な依頼の方を優先する
